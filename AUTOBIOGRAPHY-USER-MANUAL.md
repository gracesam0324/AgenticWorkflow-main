# AI Autobiography Generator 사용자 매뉴얼

> 인터뷰에서 출판 품질 자서전까지 — 이 매뉴얼은 **자서전을 만드는 가장 쉬운 방법**부터 안내합니다.

## 1. "시작하자" — 가장 쉬운 사용법

### 1.1 설치 (1회)

```bash
npm install -g @anthropic-ai/claude-code    # Claude Code CLI
pip install pyyaml jsonschema               # Python 의존성
```

### 1.2 실행 (매번)

```bash
cd Writingbook-AgenticWorkflow
claude
```

```
> 시작하자
```

**이것이 전부입니다.** `시작하자`를 입력하면 다음이 순서대로 일어납니다:

```
┌─────────────────────────────────────────────────────────────────┐
│  "시작하자" 입력                                                 │
│      ↓                                                          │
│  1. 실행 모드 선택 (Manual / Autopilot / Autopilot+ULW)          │
│      ↓                                                          │
│  2. 주인공 이름 입력                                              │
│      ↓                                                          │
│  3. 프로젝트 자동 초기화 (projects/20260318-이름/)                │
│      ↓                                                          │
│  4. Step 1~12 자동 진행 시작                                      │
│      ↓                                                          │
│  [AI 인터뷰 12회] → [스토리 바이블] → [챕터 산문] → [PDF/EPUB]    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 실행 모드 선택 화면

```
━━━ 실행 모드 선택 ━━━
  [1] Manual     — 모든 단계에서 사용자 확인 (처음 사용자 추천)
  [2] Autopilot  — (human) 단계 자동 승인, 4계층 QA가 품질 보장
  [3] Autopilot + ULW — 최고 품질 추구 (Sisyphus Persistence)

> 2

주인공 이름을 입력하세요: 최윤식

✅ 프로젝트 초기화 완료
   ID: 20260318-최윤식
   디렉터리: autobiography-generator/projects/20260318-최윤식
   Step 1: Interview Planning 시작...
```

| 모드 | 설명 | 추천 대상 |
|------|------|---------|
| **Manual** | 모든 단계에서 사용자 확인을 요청 | 처음 사용자, 세밀한 제어 필요 시 |
| **Autopilot** | Step 3, 7, 11의 (human) 단계를 자동 승인 | 숙련 사용자 |
| **Autopilot + ULW** | Autopilot + 철저함 오버레이 (최대 3회 재시도) | 최고 품질 추구 |

> **트리거**: "시작", "시작하자", "시작하라", "시작해", "start", "begin", "let's start" 모두 동일하게 작동합니다.

### 1.4 중단 후 재개

세션이 끊기거나 `/clear`한 경우, 다시 `시작하자`를 입력하면 **중단된 단계부터 자동 재개**됩니다. `state.yaml`이 현재 진행 상태를 기억합니다.

## 3. Research 단계 (Steps 1-4)

### 3.1 Step 1: 인터뷰 계획 수립

Orchestrator가 인물의 인생 시기와 테마를 분석하여 인터뷰 계획을 수립합니다.

**산출물**: `outputs/interviews/interview_plan.md`

내용:
- 인생 시기 매핑 (출생~현재)
- 최소 5개 인터뷰 테마 식별
- 세션 스케줄 (주요 인생 시기별 1세션)
- 최소 3세션 계획

### 3.2 Step 2: 인터뷰 수행

`@interviewer` 에이전트가 대화형 인터뷰를 진행합니다.

**인터뷰 진행 방식**:

1. **세션 시작**: 에이전트가 인생 시기/테마에 대한 개방형 질문으로 대화 시작
2. **심층 탐구**: 감각적 디테일(소리, 냄새, 장면), 정확한 대화, 감정 반응을 끌어냄
3. **세션 완료**: 매 세션 후 다음이 표시됩니다:

```
──────── 세션 요약 ────────
테마: 어린 시절, 가족 관계
주요 인용: "어머니는 항상 새벽 4시에 일어나셨다"
등장 인물: 어머니, 외할머니, 담임 선생님
장소: 충주 시골집, 초등학교
감정 하이라이트: 이사 날의 눈물

──────── 커버리지 맵 ────────
┌─────────────────┬──────────┬─────────┐
│ Life Period      │ Sessions │ Depth   │
├─────────────────┼──────────┼─────────┤
│ Early Childhood  │ 1/1      │ ████░   │
│ Late Childhood   │ 0/0      │ ---     │
│ Adolescence      │ 0/0      │ ---     │
│ ...              │          │         │
└─────────────────┴──────────┴─────────┘

Q1: "이번 세션에서 더 깊이 탐구하고 싶은 부분이 있습니까?"
Q2: "다음 세션은 어떤 시기/주제에 집중할까요?"
Q3: "인터뷰를 마무리할 준비가 되셨나요, 아니면 계속할까요?"
```

**중요 규칙**:
- 최소 3세션 필수
- 매 세션 후 사용자 확인 없이 절대 다음으로 진행하지 않음
- 사용자가 "마무리"를 명시적으로 확인해야 Step 3으로 진행

**산출물**: `outputs/interviews/INT-{NNN}.json` (세션별)

### 3.3 Step 3: (human) 인터뷰 검토

인터뷰 세트의 완성도를 검토합니다.

**화면에 표시되는 정보**:
- 커버리지 맵 (인생 시기별 세션 수 + 깊이)
- 소재 평가 (`assess_material.py` 결과)
- 인터뷰 요약 (테마, 핵심 인용, 등장 인물)

**선택지**:
1. **승인** → Step 4로 진행
2. **추가 세션 요청** → 특정 주제 지정 후 Step 2로 복귀
3. **녹취 수정** → 사실 오류 교정

> **Autopilot 모드**: 이 단계는 자동 승인됩니다 (커버리지가 충분하면 승인).

### 3.4 Step 4: (human) 스토리 블루프린트 공동 창작

**이 단계가 가장 중요합니다.** 사용자와 AI가 함께 자서전의 서사 구조를 결정합니다.

#### Phase A: 인사이트 제시 (자동)

`@story-architect`가 인터뷰를 분석하여 3개 후보 구조를 제안합니다:

```
━━━ 구조 A: 시간순 ━━━
  출생 → 성장 → 경력 → 현재
  12개 챕터 제목 초안
  강점: 자연스러운 흐름, 독자가 따라가기 쉬움
  약점: 드라마틱한 전환점이 묻힐 수 있음

━━━ 구조 B: 테마별 ━━━
  가족 / 경력 / 신앙 클러스터
  12개 챕터 제목 초안
  강점: 주제의 깊이, 관련 경험 집중
  약점: 시간의 흐름을 느끼기 어려움

━━━ 구조 C: 킥포인트 중심 ━━━
  3-5개 인생 전환점을 앵커로
  12개 챕터 제목 초안
  강점: 극적 서사, 읽는 재미
  약점: 전환점 사이의 일상이 약해질 수 있음
```

각 구조에 포함되는 요소:
- 12개 챕터 제목 초안
- 3-5개 킥포인트 (인생 전환점)
- 2개 중심 메타포 후보
- 감정 아크 시각화 (한-흥-정 사이클)
- 소재 강도 맵 (풍부한 영역 vs 빈약한 영역)

#### Phase B: 사용자 협의 (핵심 상호작용)

```
Q1: "어떤 구조가 가장 마음에 드시나요? 다른 비전이 있으신가요?"
Q2: "인생에서 가장 중요한 전환점 3가지는 무엇인가요?"
Q3: "독자가 이 책에서 가져가야 할 단 하나의 메시지는 무엇인가요?"
Q4: "특별히 강조하거나 빼고 싶은 내용이 있나요?"
```

모든 응답이 `story_blueprint.json`에 기록됩니다.

#### Phase C: 블루프린트 확정 (사용자 승인)

```
━━━ 최종 블루프린트 ━━━
  구조: 킥포인트 중심 (사용자 선택)
  킥포인트: [결혼, 이직, 암 진단]
  핵심 메시지: "하나님의 인도하심..."
  강조 영역: [가족, 신앙]
  배제 영역: [법적 분쟁]
  중심 메타포: "나뭇결 위의 지도"

  [승인] / [수정 요청] / [처음부터 재설계]
```

**산출물**: `outputs/story-blueprint/story_blueprint.json`

> **이 블루프린트는 이후 모든 단계를 구속합니다.** 스토리 바이블, 챕터 플랜, 산문 작성이 모두 이 결정을 따릅니다.

## 4. Planning 단계 (Steps 5-8)

### 4.1 Step 5: 스토리 바이블 구축

`@story-architect`가 모든 인터뷰 데이터를 종합하여 자서전의 **단일 진실 소스(SOT)**인 스토리 바이블을 구축합니다.

**스토리 바이블 구성요소**:

| 섹션 | 내용 | 예시 |
|------|------|------|
| characters | 인물 레지스트리 (정규 이름, 별칭, 관계, 등장 챕터) | 최윤식(주인공), 이숙희(아내), ... |
| timeline | 크로노로지컬 이벤트 (날짜, 중요도, 출처) | 1983-03: 대학 입학 |
| places | 장소 레지스트리 (위치, 시기, 감각 디테일) | 충주 시골집: "마당의 감나무 냄새" |
| themes | 테마 분석 (5-10개, 관련 인터뷰 참조) | 신앙 여정, 직업 소명, ... |
| chapters | 챕터 플랜 (블루프린트 구속) | Ch.1: "나뭇결 위의 지도" 1965-1976 |
| voice_guide | 보이스 가이드 (Python 계산 메트릭 기반) | avg_sentence_length: 14.2 |
| fact_registry | 검증 가능한 사실 목록 | "1989년 한양대 입학" |

**검증**: `validate_story_bible.py`가 SB1-SB10 체크를 자동 실행합니다.

### 4.2 Step 6: 스토리 바이블 리뷰

`@reviewer`가 스토리 바이블을 적대적으로 리뷰합니다.

**리뷰 관점**:
- 사실 정확성: 인터뷰와 대조 (CoVe 프로토콜)
- 타임라인 일관성: 날짜 모순 탐지
- 캐릭터 완성도: 주요 인물의 관계 아크
- 테마 커버리지: 모든 테마가 충분히 다뤄지는지
- 블루프린트 준수: 킥포인트, 강조/배제 영역 반영 여부

**산출물**: `review-logs/RV-story-bible.json`

### 4.3 Step 7: (human) 스토리 바이블 승인

사용자가 스토리 바이블과 리뷰어 피드백을 검토합니다.

**화면에 표시되는 정보**:
- 스토리 바이블 요약 (인물, 타임라인, 테마, 챕터 플랜)
- 리뷰어 피드백 (critical/warning/suggestion 건수)
- pACS 점수 이력

**선택지**:
1. **승인** → Step 8 (글쓰기 스타일 선택)으로 진행
2. **스토리 바이블 수정** → Step 5로 복귀
3. **챕터 플랜 재구성** → 챕터 순서/범위 수정
4. **블루프린트 수정** → Step 4로 복귀 (서사 구조 변경)

> **Autopilot 모드**: 이 단계는 자동 승인됩니다.

### 4.4 Step 8: (human) 글쓰기 스타일 선택 & 캘리브레이션

#### Phase A: 스타일 갤러리 (자동)

10종 작가 스타일이 카드 형태로 표시됩니다:

```
━━━ Style 1: Hemingway ━━━
  핵심 기법: 빙산 이론, 감정 절제
  문장 길이: 짧음 (평균 8단어)
  대화 비율: 높음
  적합한 유형: 행동 중심, 감정 절제 자서전
  골든 예시: "The old man sat alone at the table..."

━━━ Top 3 추천 ━━━
  1. Hesse (유사도 78%) — "당신의 인터뷰 보이스는 영적 탐구 패턴을 보입니다"
  2. Murakami (유사도 72%) — "일상 속 성찰적 톤이 유사합니다"
  3. Márquez (유사도 65%) — "감각적 묘사가 풍부합니다"

  💡 Custom Blend: 2-3개 스타일을 혼합할 수도 있습니다
```

#### Phase B: 사용자 선택

```
Q1: "어떤 스타일이 마음에 드시나요? (번호, 이름, 또는 'custom')"
Q2: "블렌딩 비율을 조정하시겠습니까? (기본: 30% 스타일 / 70% 당신의 목소리)"
Q3: "특별히 강조하거나 피하고 싶은 스타일 요소가 있나요?"
```

#### Phase C: 캘리브레이션 샘플 (자동 → 사용자)

`@chapter-writer`가 Chapter 1 오프닝 (~500단어)을 3가지 버전으로 작성합니다:

```
Version A: 스타일 강조 (50% 스타일 / 50% 보이스)
Version B: 균형 블렌드 (30% 스타일 / 70% 보이스) — 기본
Version C: 보이스 강조 (10% 스타일 / 90% 보이스)
```

사용자가 선호하는 버전을 선택하면 블렌딩 비율이 확정됩니다.

#### Phase D: 보이스 가이드 확정 (자동)

`style_blender.py`가 최종 블렌딩 파라미터를 계산합니다 (순수 산술). 스토리 바이블의 `voice_guide`가 블렌딩된 메트릭으로 업데이트됩니다.

**산출물**: `outputs/style-selection/style_selection.json` + 3개 캘리브레이션 샘플

## 5. Implementation 단계 (Steps 9-12)

### 5.1 Step 9: 챕터 작성 루프

각 챕터에 대해 4단계 서브루프가 실행됩니다:

#### 9a. 컨텍스트 조립

`assemble_chapter_context.py`가 해당 챕터에 필요한 정보만 필터링합니다:
- 관련 인물, 이벤트, 장소 (스토리 바이블에서 추출)
- 해당 챕터의 소스 인터뷰 세그먼트
- 이전 챕터 마지막 단락 (연속성)
- 블렌딩된 보이스 가이드 수치

#### 9b. 챕터 초안 작성

`@chapter-writer`가 문학적 산문을 작성합니다.

**작성 원칙**:
- **Show, Don't Tell**: 감정을 직접 서술하지 않고 장면으로 보여줌
- **보이스 가이드 준수**: 블렌딩된 수치 목표를 따름
- **소스 참조**: 인터뷰 인용/이벤트 출처 태깅
- **연속성**: 이전 챕터와 자연스러운 전환

**산출물**: `outputs/chapters/ch{NN}_draft_v{V}.md` + `.meta.json`

#### 9c. 보이스 캘리브레이션

`@voice-calibrator`가 챕터의 보이스 메트릭을 측정합니다:

```
━━━ Voice Calibration Report ━━━
  문장 길이: 15.3 (목표: 14.2 ± 2.0) ✅
  대화 비율: 0.28 (목표: 0.32 ± 0.05) ⚠ 미달
  수동태: 7% (목표: < 15%) ✅
  부사 밀도: 14.1/1000 (목표: < 20) ✅
  Show:Tell: 0.72 (목표: > 0.60) ✅
  금지어: 0건 ✅
```

`validate_voice_match.py` (VM1-VM6)가 결정론적으로 최종 검증합니다.

#### 9d. 챕터 품질 게이트

`@reviewer`가 7차원 적대적 리뷰를 수행합니다:

```
━━━ Chapter Review Verdict ━━━
  감정적 공명:    85/100  (한-흥-정 전환 2회, 여운 엔딩 ✅)
  보이스 일관성:  90/100  (골든 예시 패턴 매치)
  산문 품질:      82/100  (번역체 없음, 침묵 1건)
  사실 정확성:    88/100  (CoVe 12/13 검증, 1건 [INFERRED])
  서사 아크:      78/100  (기승전결 ✅, 의미 발견 1건)
  출처 추적성:    92/100  (11/12 출처 확인)
  연속성:         95/100  (이전 챕터 스레드 유지)

  종합: 87/100 — PASS
  개선 제안: "대화 비율을 10% 높이면 장면의 생동감이 올라갑니다"
```

- **PASS**: 다음 챕터로 진행 (+ `@translator` 한국어 번역)
- **FAIL**: Step 9b로 복귀 (피드백 반영 수정, 최대 3-5라운드)

### 5.2 Step 10: 교차 챕터 일관성 검사

전체 원고에 대한 종합 일관성 감사입니다.

**Agent Team 구성** (차원별 분할):
- `@reviewer`: 이름/캐릭터 일관성 (CC1-CC3)
- `@reviewer`: 타임라인/장소 일관성 (CC4-CC6)
- `@reviewer-deep`: 반복 서사 / 동질화 탐지 (CC7-CC8)

**검증 항목 예시**:
- Ch.5에서 "이숙희"로 불린 인물이 Ch.8에서 "숙희씨"로 바뀌지 않았는지

**산출물**: `review-logs/consistency-report.json`

## 8. 품질 관리 이해하기

### 8.1 pACS 점수 읽기

pACS(performative Agent Competency Score)는 에이전트가 자기 작업의 품질을 3차원으로 평가한 점수입니다:

| 차원 | 의미 | 약점 시 대응 |
|------|------|------------|
| **F** (Factual) | 사실 정확성 | 인터뷰 재확인 필요 |
| **C** (Creative) | 창의성/산문 품질 | 스타일 캘리브레이션 재조정 |
| **L** (Logical) | 논리적 일관성 | 스토리 바이블 교차 참조 강화 |

**점수 해석**:
- 90+ : 우수
- 80-89 : 양호
- 70-79 : 개선 필요
- 60 미만 : 해당 단계 재실행 필요

`validate_pacs_floor.py`가 자기 과대평가를 검증합니다 — 에이전트가 점수를 부풀리면 탐지됩니다.

### 8.2 리뷰 피드백 읽기

`review-logs/RV-*.json`의 리뷰 판정은 다음 구조를 따릅니다:

```json
{
  "verdict": "PASS" | "FAIL" | "CONDITIONAL_PASS",
  "dimensions": {
    "emotional_resonance": {"score": 85, "feedback": "..."},
    "voice_consistency": {"score": 90, "feedback": "..."},
    ...
  },
  "issues": [
    {"severity": "critical", "description": "...", "location": "Ch.3 para.5"},
    {"severity": "warning", "description": "...", "location": "Ch.3 para.12"},
    {"severity": "suggestion", "description": "...", "location": "..."}
  ],
  "pre_mortem": "...",
  "pacs_independent": {"F": 80, "C": 85, "L": 88}
}
```

| 심각도 | 의미 | 대응 |
|--------|------|------|
| `critical` | 출판 불가 결함 | 반드시 수정 후 재리뷰 |
| `warning` | 품질 저하 위험 | 가능하면 수정 |
| `suggestion` | 개선 제안 | 선택적 반영 |

## 11. 한국 문학 품질 시스템 이해하기

이 시스템이 생산하는 자서전은 단순한 기록이 아니라 **문학 작품**입니다. 11개의 문학 지시문(Appendix A)이 글쓰기와 리뷰 과정을 지배합니다.

### 11.1 핵심 지시문 요약

| 지시문 | 규칙 | 사용자가 알아야 할 것 |
|--------|------|-------------------|
| **한/흥/정** (A-6) | 감정을 라벨이 아닌 장면으로 전달 | 리뷰에서 "한/흥/정이 체화되지 않음" 피드백 = 감정 장면 부족 |
| **기승전결** (A-5) | 전(轉) = 관점 전환, 클라이맥스 아님 | 챕터 구조가 기-승-전-결 리듬을 따르는지 확인 |
| **여운** (A-3) | 교훈/도덕/요약 없는 엔딩 | 챕터가 "그래서 나는 ~을 배웠다"로 끝나면 수정 필요 |
| **침묵** (A-4) | 침묵을 서사 콘텐츠로 사용 | 챕터당 최소 1회 침묵이 내용으로 쓰인 장면 |
| **번역체 금지** (A-10) | 9가지 한국어 번역체 패턴 차단 | Python이 패턴 5-9 자동 탐지, 리뷰어가 패턴 1-4 질적 평가 |
| **이중 의식** (A-7) | 인생 시기별 체험:서술 비율 | 어린 시절 챕터에 성찰이 많으면 비율 위반 |
| **장면 6단계** (A-9) | 핫스팟→장소→감각→대화→신체화→의미 | 챕터당 완전 구성 장면 ≥2개 |
| **[INFERRED]** (A-11) | 윤색 요소 명시적 태깅 | 메타데이터에서 어떤 요소가 추론인지 투명하게 확인 |

### 11.2 리뷰 피드백에서 자주 보이는 문학 관련 이슈

| 이슈 | 의미 | 대응 |
|------|------|------|
| "번역체 패턴 감지" | 한국어가 영어 번역문처럼 읽힘 | 9패턴 중 해당 패턴 수정 |
| "여운 미달" | 챕터가 교훈/요약으로 끝남 | 4가지 허용 엔딩 기법 중 하나로 교체 |
| "Golden Exemplar 부재" | 특별히 생동감 있는 구절이 없음 | 인터뷰 핫스팟 활용도 높이기 |
| "동질화 위험" | 챕터 간 구조/패턴 반복 | 텍스처, 오프닝/클로징 패턴 다양화 |
| "한/흥/정 라벨 사용" | 감정 단어를 직접 서술 | 체화된 장면으로 교체 |

## 13. 용어 사전

| 용어 | 설명 |
|------|------|
| **SOT** | Single Source of Truth — 모든 에이전트가 참조하는 단일 진실 소스 |
| **pACS** | performative Agent Competency Score — 에이전트 자기 품질 평가 (F/C/L 3차원) |
| **CoVe** | Chain-of-Verification — 사실 주장을 추출하여 원본 대조 검증하는 프로토콜 |
| **킥포인트** | 인생의 주요 전환점 (자서전 서사의 앵커) |
| **블루프린트** | 사용자와 합의한 서사 구조 설계도 (이후 모든 단계를 구속) |
| **스토리 바이블** | 캐릭터·타임라인·테마·챕터 플랜 등을 포함한 서사 SOT |
| **보이스 가이드** | 화자의 고유 보이스를 수치로 정의한 메트릭 세트 |
| **보이스 지문** | 인터뷰에서 Python으로 추출한 결정론적 보이스 파라미터 |
| **스타일 블렌딩** | 작가 스타일과 화자 보이스를 수학적으로 혼합 |
| **한-흥-정** | 한국 서사 전통의 감정 구조 (恨-興-情 사이클) |
| **Human Gate** | 사용자의 판단이 필수인 워크플로우 체크포인트 |
| **Agent Team** | 복수 에이전트가 병렬로 작업하는 팀 구성 |
| **[INFERRED]** | 인터뷰에 명시되지 않은 AI의 추론임을 표시하는 태그 |
| **L0-L2** | 4계층 품질 보장 체계 (Anti-Skip → Verification → pACS → Review) |
| **P1** | 할루시네이션 봉쇄 원칙 — 결정론적 Python 검증으로 AI 날조 방지 |
| **DNA 유전** | 부모 프레임워크(AgenticWorkflow)의 게놈을 자식 시스템이 구조적으로 상속 |
| **기승전결** | 한국 서사의 4막 구조 (起承轉結) — 전(轉)은 관점 전환이지 클라이맥스가 아님 |
| **여운** | 챕터 엔딩 기법 — 교훈/요약 없이 여운을 남기는 4가지 허용 기법 |
| **번역체** | AI 생성 한국어 텍스트의 흔한 결함 — 영어 구문이 한국어에 투영된 패턴 |
| **Golden Exemplar** | 인터뷰에서 추출한 비범하게 생동감 있는 3-5개 구절 (톤 앵커) |
| **이중 의식** | 체험하는 자아(과거) vs 서술하는 자아(현재)의 비율 — 인생 시기별 조절 |
| **DC-1~DC-7** | @reviewer-deep의 7개 동질화 방지 체크 (반복 일화, 패턴, 텍스처, 메타포, 거울, 예시, 레지스터) |
| **Appendix A** | 11개 한국 문학 품질 지시문 (A-1~A-11) — @chapter-writer와 @reviewer에 비협상적 의무 |
