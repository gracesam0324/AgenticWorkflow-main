# AI Studio 사용자 매뉴얼

> 이 매뉴얼은 AI Studio를 **처음 사용하는 분**을 위해 작성되었습니다.
> 코딩 경험이 없어도 괜찮습니다. 한국어로 대화하면 됩니다.

---

## 1. 시작하기 — 가장 쉬운 방법

Claude Code를 열고, 다음 중 아무거나 입력하세요:

```
시작하자
```

이것만으로 충분합니다. 다른 표현도 가능합니다:

| 입력 예시 | 모두 같은 결과 |
|----------|--------------|
| 시작 | 메뉴가 나타남 |
| 시작하자 | 메뉴가 나타남 |
| 시작해 | 메뉴가 나타남 |
| start | 메뉴가 나타남 |
| 작업을 시작하자 | 메뉴가 나타남 |

### 메뉴가 나타나면

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   AgenticWorkflow — 제품 실행 모드
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 수련회 앱 만들기
  2. 자서전 쓰기
  3. 워크플로우 설계
  4. 학술 글쓰기
```

**번호를 입력**하거나, **하고 싶은 것을 그냥 말하세요**:

```
1
```

또는:

```
수련회 앱 만들어줘
```

또는:

```
교회 청소년부 수련회에서 쓸 성경 퀴즈 앱이 필요해요
```

세 가지 모두 같은 곳으로 안내됩니다. 편한 방식으로 입력하세요.

## 2. 수련회 앱 만들기

### 대화 흐름

```
나: "시작하자"
AI: [메뉴 표시]
나: "1" (또는 "수련회 앱")

AI: "어떤 앱을 만들까요?"
    1. 성경 퀴즈 — 팀별 실시간 대결!
    2. 스탬프 랠리 — QR코드로 미션 클리어!
    3. 일정 및 공지 — 실시간 일정표
    4. 찬양 가사 — 실시간 가사 표시
    5. QT 가이드 — 매일 묵상 안내
    6. 팀 점수판 — 실시간 점수 경쟁
    7. 포토 갤러리 — 추억 공유
    8. 기도 제목 — 익명 기도 나눔
    9. 통합 앱 — 위 기능 조합

나: "성경 퀴즈"

AI: "어떤 성경 범위를 다루나요?"
나: "마태복음이요"

AI: "몇 팀이 참가하나요?"
나: "4팀이요"

AI: "퀴즈 문제를 직접 넣으실 건가요, AI가 만들까요?"
나: "AI가 만들어줘"

... (AI가 앱을 생성합니다) ...

AI: "앱이 완성되었습니다! 확인해 보세요."
    [브라우저에서 앱이 열림]
    [QR코드가 표시됨]

나: "글씨가 좀 작은데 키워줘"
AI: → 수정 → "수정했습니다. 확인해 보세요."

나: "완성이에요!"
AI: → 배포 완료. QR코드 + 실행 파일 생성
```

### 최종 결과물

| 결과물 | 설명 |
|--------|------|
| 모바일 웹앱 | 브라우저에서 바로 실행되는 앱 |
| QR코드 (PNG) | 학생들이 스마트폰으로 스캔하여 접속 |
| 실행 파일 (.bat) | 더블클릭으로 서버 시작 |
| WiFi 안내 페이지 | 접속 방법을 알려주는 페이지 |

## 3. 자서전 쓰기

### 대화 흐름

```
나: "시작하자"
AI: [메뉴 표시]
나: "2" (또는 "자서전 쓰고 싶어요")

AI: "자서전 인터뷰를 시작하겠습니다.
     편하게 이야기해 주세요.
     어린 시절부터 시작해 볼까요?"

나: "초등학교 때 시골에서 자랐어요..."
AI: "그때 가장 기억에 남는 순간이 있나요?"
나: "전학 갔을 때 처음으로 친구를 사귄 이야기요..."

... (여러 세션에 걸쳐 인터뷰가 진행됩니다) ...

AI: "인터뷰 자료가 충분합니다.
     이제 책의 구조를 설계하겠습니다."
    [아웃라인 제시]

나: "3장의 순서를 바꿔주세요"
AI: → 수정 → "확인해 주세요"
나: "좋아요"

... (AI가 챕터별로 집필합니다) ...

AI: "1장 초안입니다. 읽어보시고 의견 주세요."
나: "좋은데, 마지막 단락이 좀 딱딱해요"
AI: → 수정 → "수정했습니다"

... (모든 챕터 완성 후) ...

AI: "PDF와 EPUB 파일이 준비되었습니다."
```

### 최종 결과물

| 결과물 | 설명 |
|--------|------|
| PDF 파일 | 인쇄 가능한 자서전 |
| EPUB 파일 | 전자책 리더용 |
| 원고 파일 (.md) | 편집 가능한 텍스트 |

### 중간에 멈추고 나중에 이어하기

작업을 중간에 멈춰도 괜찮습니다. 다음에 다시 "시작하자"를 입력하면:

```
AI: "자서전 쓰기 [진행 중] — Step 7: 챕터 집필"
    "이어서 진행할까요, 새로 시작할까요?"

나: "이어서"
AI: → 이전 작업을 이어서 진행합니다
```

### 대화 흐름

```
나: "시작하자"

AI: "어떤 작업을 자동화하고 싶으세요?"

AI: "몇 가지 질문이 있습니다..."
    1. 블로그 주제는 어떻게 결정되나요?
    2. 글의 길이와 스타일은?
    3. 게시 전에 사람이 검토하나요?
    ...

... (대화를 통해 요구사항 파악) ...

AI: "워크플로우 설계가 완성되었습니다."
    [workflow.md 파일 생성]
```

### 최종 결과물

| 결과물 | 설명 |
|--------|------|
| workflow.md | 자동화 워크플로우 설계도 |

## 6. 유용한 명령어 모음

### 언제든 사용할 수 있는 명령어

| 입력 | 동작 |
|------|------|
| **시작하자** | 실행 모드 메뉴 표시 |
| **/status** | 현재 프로젝트 상태 확인 |
| **/maintenance** | 시스템 건강 검진 |

### 수련회 앱 전용 명령어

| 입력 | 동작 |
|------|------|
| **/start-app** | 새 앱 만들기 시작 |
| **/resume-app** | 이전 앱 이어서 만들기 |
| **/deploy-app** | 앱 배포 (QR코드 생성) |
| **/app-status** | 앱 진행 상태 확인 |
| **/app-verify** | 앱 품질 검증 |

### 자서전 전용 명령어

| 입력 | 동작 |
|------|------|
| **/interview** | 인터뷰 시작/계속 |
| **/review-chapter** | 챕터 검토 |
| **/export** | PDF/EPUB 내보내기 |
| **/build-verify** | 빌드 검증 |

## 7. 문제가 생겼을 때

### "아까 하던 작업이 어디 갔지?"

걱정하지 마세요. 작업 내역은 **자동으로 저장**됩니다.
"시작하자"를 다시 입력하면 이전 작업을 이어갈 수 있습니다.

### "뭔가 이상하게 동작해요"

```
/maintenance
```

를 입력하면 시스템이 스스로 건강 검진을 수행하고 문제를 알려줍니다.

### "처음부터 다시 시작하고 싶어요"

메뉴에서 모드를 선택한 후, "새로 시작할까요?" 질문에 "새로"를 선택하세요.

## 8. 알아두면 좋은 것들

### 이 시스템이 하는 일

- **대화를 통해 의도를 파악**합니다 — 기술적 용어를 몰라도 됩니다
- **자동으로 품질을 검증**합니다 — 4단계 검증 시스템이 내장되어 있습니다
- **작업 내역을 자동 저장**합니다 — 세션이 끊어져도 복원됩니다
- **위험한 작업을 차단**합니다 — 131개 안전 테스트가 동작합니다

### 이 시스템이 하지 않는 일

- 인터넷에 개인정보를 전송하지 않습니다
- 사용자 동의 없이 파일을 삭제하지 않습니다
- 빌드/인프라 설정을 사용자에게 요구하지 않습니다
