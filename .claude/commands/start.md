# /start — 제품 실행 모드 진입점

> 사용자가 "시작", "시작하자", "start", "워크플로우를 시작하자", "작업을 시작하자" 등
> 시작을 알리는 자연어를 입력하면 이 커맨드가 실행된다.

---

## Step 0: 프로젝트 감지 + 메뉴 렌더링

**Bash로 실행** (사용자에게 과정을 보여주지 않는다):

```bash
python3 "$CLAUDE_PROJECT_DIR/.claude/hooks/scripts/detect_projects.py" \
  --project-dir="$CLAUDE_PROJECT_DIR" --render=menu
```

> Windows에서 `python3`가 실패하면 `py` 또는 `python`으로 재시도.

스크립트가 마크다운 형식의 **메뉴 전체**를 stdout으로 출력한다.
이 출력을 **그대로** 사용자에게 보여준다. 수정하지 않는다.

스크립트가 실패하�� Step 0F(Fallback)로 이동한다.

## 절대 금지 규칙 (NEVER SHOW)

> **"Infrastructure Build", "인프라 빌드", "빌드 재실행", "프레임워크 설치", "초기 설정", "setup", "install"**
> 과 관련된 옵션은 **어떤 형태로든 절대 메뉴에 표시하지 않는다.**
> 이 규칙은 예외 없이 적용된다. 사용자가 명시적으로 요청하더라도 이 메뉴에서는 표시하지 않는다.

## Step 1: 사용자 선택 대기 및 라우팅

사용자 입력을 받으면 아래 매핑 규칙에 따라 라우팅한다:

| 사용자 입력 (예시) | 매핑 스킬 | 실행 액션 |
|-------------------|----------|----------|
| `1`, "수련회", "앱", "교회", "retreat" | `church-retreat-app` | → Step 2A |
| `2`, "자서전", "내 이야기", "autobiography" | `autobiography` | → Step 2B |
| `3`, "워크플로우", "자동화", "workflow" | `workflow-generator` | → Step 2C |
| `4`, "논문", "학술", "글쓰기", "academic" | `doctoral-writing` | → Step 2D |

**모호한 입력**: 위 키워드에 해당하지 않으면, P4 규칙에 따라 최대 2개 질문으로 의도를 파악한다.

---

## Step 2A: 수련회 앱 모드 진입

1. **진행 중 감지**: `app-state.json`이 존재하고 `current_step > 0`이면:
   - 현재 상태를 한 줄로 요약 (예: "Phase 3 코드 생성 중, 진행률 43%")
   - 질문: **"이어서 진행할까요, 새로 시작할까요?"**
   - 이어서 → `/resume-app` 실행
   - 새로 → `/start-app` 실행

2. **신규**: `/start-app` 실행

---

## Step 2B: 자서전 모드 진입

1. **진행 중 감지**: `autobiography-generator/state.yaml`이 존재하고 `current_step > 0`이면:
   - 현재 상태를 한 줄로 요약 (예: "Step 7 챕터 집필 중, 진행률 58%")
   - 질문: **"이어서 진행할까요, 새로 시작할까요?"**
   - 이어서 → 현재 단계에 맞는 커맨드 실행 (`/interview`, `/review-chapter` 등)
   - 새로 → `/interview` 실행

2. **신규**: `/interview` 실행

---

## Step 2C: 워크플로우 설계 모드 진입

`workflow-generator` 스킬을 직접 실행한다:
- 스킬 진입점: `.claude/skills/workflow-generator/SKILL.md`를 Read하여 프로토콜에 따라 대화 시작

---

## Step 2D: 학술 글쓰기 모드 진입

`doctoral-writing` 스킬을 직접 실행한다:
- 스킬 진입점: `.claude/skills/doctoral-writing/SKILL.md`를 Read하여 프로토콜에 따라 대화 시작

| 상황 | 동작 |
|------|------|

## 제약

- **English-First 실행 (AC-4)**: 내부 판단은 영어로 수행. 사용자 대면 출력만 한국어.
- **절대 기준 1 (품질)**: 메뉴 UI는 사용자가 직관적으로 이해할 수 있어야 한다.
- **P4 (질문 설계)**: 사용자에게 질문할 때 최대 4개, 각 3개 선택지.
- **NEVER SHOW Infrastructure Build**: 이 커맨드의 메뉴에서 빌드/설치/인프라 관련 옵션은 절대 표시하지 않는다.
