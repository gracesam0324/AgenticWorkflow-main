## 1. 3단계 자율성 레벨 UX

### Level 1: 완전 수동 (Supervised Mode)

모든 스텝마다 y/n/s(skip)/i(inspect) 입력 요구.

```
╔══════════════════════════════════════════════════════════════════╗
║  autoflow  v0.1.0          Mode: SUPERVISED  [●  확인 필요]     ║
╚══════════════════════════════════════════════════════════════════╝

 Task: "logs 디렉토리에서 30일 이상 된 파일을 압축 후 archive/ 로 이동"

 ┌─ Plan Preview ──────────────────────────────────────────────────┐
 │  Steps: 5    Est. Time: ~2m    Rollback: 가능    Risk: LOW      │
 └─────────────────────────────────────────────────────────────────┘

 ──────────────────────────────────────────────────────────────────
 STEP 1 / 5  ▸  파일 목록 스캔
 ──────────────────────────────────────────────────────────────────
  Action   : find /var/logs -mtime +30 -name "*.log"
  Type     : READ ONLY
  Risk     : ● LOW
  Rollback : 불필요 (읽기 전용)
  Preview  :
    └─ 영향 경로: /var/logs/** (읽기만)

  [y] 실행   [n] 중단   [s] 이 스텝 건너뜀   [i] 상세 보기
  > _
```

### Level 2: 반자율 (Semi-Autonomous Mode)

**"안전" 기준**: 읽기 전용, 임시 디렉토리 접근, 네트워크 없음, 기존 파일 비변경
**"위험" 기준**: 파일 삭제/덮어쓰기, 네트워크 요청, 외부 프로세스 실행, 환경 변수 변경

```
╔══════════════════════════════════════════════════════════════════╗
║  autoflow  v0.1.0       Mode: SEMI-AUTO  [◐  선택적 확인]       ║
╚══════════════════════════════════════════════════════════════════╝

  ⟳ STEP 1 자동 실행 중...  파일 목록 스캔 (READ ONLY)
  ✔ STEP 1 완료  (0.3s)   Found: 47 files

  ⟳ STEP 2 자동 실행 중...  임시 디렉토리 생성 (WRITE /tmp)
  ✔ STEP 2 완료  (0.1s)   Created: /tmp/autoflow_work_8a3f/

 ──────────────────────────────────────────────────────────────────
 STEP 3 / 5  ▸  원본 파일 삭제  ← 위험 스텝: 확인 필요
 ──────────────────────────────────────────────────────────────────
  Action   : rm [47 files] in /var/logs/
  Type     : DELETE — 복구 불가
  Risk     : ● HIGH

  ⚠  이 작업은 되돌릴 수 없습니다. 계속하시겠습니까?
  [y] 삭제 실행   [n] 중단   [s] 건너뜀   [i] 전체 목록 보기
  > _
```

### Level 3: 완전 자율 (Autonomous Mode)

허용 목록(allowlist)에 등록된 명령 패턴만 자동 실행. 목록 외 작업은 자동 BLOCK.

```
╔══════════════════════════════════════════════════════════════════╗
║  autoflow  v0.1.0        Mode: AUTONOMOUS  [● 자동 실행 중]     ║
║  Audit Log: ~/.autoflow/audit/2026-04-07T14:32:11.log           ║
╚══════════════════════════════════════════════════════════════════╝

  ✔  [14:32:11]  STEP 1  파일 목록 스캔           (0.3s)  ALLOWED
  ✔  [14:32:12]  STEP 2  임시 디렉토리 생성        (0.1s)  ALLOWED
  ✔  [14:32:12]  STEP 3  tar 압축                 (8.4s)  ALLOWED
  ✔  [14:32:21]  STEP 4  archive/ 로 이동         (1.2s)  ALLOWED
  ✗  [14:32:22]  STEP 5  원본 파일 삭제                   BLOCKED

 ──────────────────────────────────────────────────────────────────
 ⛔  AUTONOMOUS BLOCK: 허용 목록 외 작업 감지
 ──────────────────────────────────────────────────────────────────
  [a] allowlist에 추가 후 재실행
  [m] 이 스텝만 수동 확인으로 전환
  [s] 이 스텝 건너뜀 (나머지 계속)
  [q] 전체 중단 + 완료된 스텝 롤백
  > _

 Rollback Snapshot: .autoflow/snapshots/20260407_143211/
  └─ 복구 명령: autoflow rollback 20260407_143211
```

## 2. 단계별 승인 UX 상세 설계

### 플랜 미리보기 화면

```
 ┌─ 스텝 목록 ─────────────────────────────────────────────────────┐
 │  #  │ 작업                    │ 타입   │ 위험  │ 롤백   │ 시간  │
 │─────┼─────────────────────────┼────────┼───────┼────────┼───────│
 │  1  │ 로그 파일 탐색          │ READ   │ ● LOW │  -     │  ~2s  │
 │  2  │ 에러 패턴 파싱 (Python) │ EXEC   │ ● LOW │  -     │ ~30s  │
 │  3  │ 결과 임시 저장          │ WRITE  │ ● MED │  가능  │  ~1s  │
 │  4  │ CSV 포맷 변환           │ EXEC   │ ● LOW │  -     │  ~5s  │
 │  5  │ output/errors.csv 저장  │ WRITE  │ ● MED │  가능  │  ~1s  │
 │  6  │ 임시 파일 정리          │ DELETE │ ● MED │  가능  │  ~1s  │
 └─────────────────────────────────────────────────────────────────┘

 [Enter] 전체 실행   [e] 스텝 편집   [y] YAML 원본 보기   [q] 취소
```

```
  ✔  STEP 2  에러 패턴 파싱          ████████████  완료  (28.3s)
  ⟳  STEP 3  결과 임시 저장          ████████░░░░  진행 중...
  ○  STEP 4  CSV 포맷 변환
  ○  STEP 5  output/errors.csv 저장
  ○  STEP 6  임시 파일 정리

 전체 진행: ██████░░░░░░  3 / 6  (50%)        경과: 0:00:32
```

### 실패 시 대응 옵션

```
 ──────────────────────────────────────────────────────────────────
  ✗  STEP 3 실패: 결과 임시 저장
 ──────────────────────────────────────────────────────────────────
  오류   : PermissionError: [Errno 13] /tmp/autoflow_8a3f/result.json

  선택:
  [r] 재시도 (동일 설정)
  [f] 경로 수정 후 재시도
  [s] 이 스텝 건너뜀 (불안정할 수 있음)
  [R] 롤백  →  완료된 스텝 되돌리기
  [q] 전체 중단 (롤백 없이)
```

## 3. 오류 상황별 UX

### API 타임아웃

```
  ⚠  API 타임아웃  (30초 초과)
   [r] 재시도 (즉시)
   [w] 60초 후 재시도
   [o] 오프라인 모드: 캐시된 유사 플랜 사용
   [q] 종료
```

### LLM이 모호한 플랜 생성

```
  ⚠  플랜 검토 필요: 모호한 스텝 감지

   STEP 4가 명확하지 않습니다: "필요에 따라 파일을 정리한다"
   문제: 어떤 파일인지 특정 안 됨, 삭제인지 이동인지 불명확

   [c] 이 스텝을 더 구체적으로 재지시 (자연어 입력)
   [d] 이 스텝 삭제
   [f] 수동으로 명령 직접 입력
   [v] YAML 직접 편집 ($EDITOR 열기)
   [q] 플랜 전체 취소, 처음부터 재입력
```

### Ctrl+C 일시정지

```
^C
  ⏸  실행 일시정지  (STEP 3 / 6 진행 중)

   [c] 실행 재개 (STEP 3부터)
   [R] 지금까지 완료된 스텝 롤백
   [s] STEP 3 건너뛰고 STEP 4부터 재개
   [q] 중단 (현재 상태 유지, 롤백 없음)

   Ctrl+C 한 번 더 → 강제 종료 (롤백 없음, 권장하지 않음)
```

## 5. 인터랙션 상태 머신

```
[IDLE] → (autoflow run) →
[PLANNING] → (API 오류) → [API_ERROR]
           → (성공) →
[PLAN_REVIEW]
  ├─ [Enter/y] → [PRE_EXECUTION] → [STEP_N_CONFIRM]
  ├─ [e] → [EDITING] → [PLAN_REVIEW]
  ├─ [c] → [PLANNING] (재지시)
  └─ [q] → [IDLE]

[STEP_N_CONFIRM] (Supervised 모드)
  ├─ [y] → [STEP_N_RUNNING]
  ├─ [s] → [STEP_N+1_CONFIRM]
  └─ [n] → [ROLLBACK]

[STEP_N_RUNNING]
  ├─ Ctrl+C → [INTERRUPTED]
  ├─ 성공 → [STEP_N+1_CONFIRM] 또는 [SUMMARY]
  └─ 실패 → [STEP_FAILED]

[STEP_FAILED]
  ├─ [r] → [STEP_N_RUNNING]
  ├─ [s] → [STEP_N+1]
  ├─ [R] → [ROLLBACK]
  └─ [q] → [SUMMARY]

[ROLLBACK] → [SUMMARY] → [IDLE]
```

| 상태 | 허용 | 금지 |
|---|---|---|
| PLANNING | Ctrl+C (취소) | 플랜 편집 |
| PLAN_REVIEW | 편집, 재지시, 실행, 취소 | 직접 명령 실행 |
| STEP_N_RUNNING | Ctrl+C (일시정지) | 즉각 종료 (데이터 손상 방지) |
| ROLLBACK | 진행 확인 | 롤백 중단 |
