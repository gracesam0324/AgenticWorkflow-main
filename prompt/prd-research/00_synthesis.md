## PRD.md 작성을 위한 5가지 핵심 결정 사항

### 결정 1: 타겟 사용자
```
v1.0: Edge Case (데이터 엔지니어, 보안 연구자, 학술 연구자)
v2.0: Mainstream (일반 개발자, 운영 담당자)

페르소나:
- 김재훈 (데이터 파이프라인): 월 40시간 절감이 목표
- Alex (침투 테스트): 보고서 자동화가 목표
- Dr. Park (학술 연구): 논문 수집 자동화가 목표
```

### 결정 2: 자동화 수준
```
3단계 자율성 레벨:
Level 1 (기본): 모든 변경 전 미리보기 + 승인 (기본값)
Level 2 (숙련): 안전 작업만 자동, 삭제/수정은 승인
Level 3 (전문가): 완전 자동 + 감사 로그 + 언제든 롤백
```

### 결정 3: 안전 원칙
```
절대 차단 (코드 레벨):
- 삭제 명령어 자동 실행
- .env / SSH key 접근 (명시 허용 전)
- 네트워크 외부 전송
- git push --force

필수 내장:
- 모든 변경 전 git commit 자동 생성
- dry-run 모드
- append-only 감사 로그
```

### 결정 4: v1.0 기술 범위
```
포함:
- 자연어 → YAML 플랜 변환
- 순차적 단계 실행
- 단일 에이전트
- 파일 기반 상태 관리
- 5-10개 큐레이션 템플릿
- 4계층 검증 (L0-L2)

제외:
- 멀티 에이전트 병렬 실행
- GUI / 웹 대시보드
- 클라우드 연동
- 커스텀 에이전트 제작
```

### 결정 5: 수익화 모델
```
Phase 1 (0-6개월): Edge Case Professional
- 개인: $300/월
- 팀 (10명 이하): $1,500/월
- 목표: 유료 고객 20-50명

Phase 2 (6-12개월): 확장
- 개인 Starter: $99/월
- Team: $499/월
- 목표: 유료 고객 200명

Phase 3 (12개월+): Enterprise
- Site License: $50,000-100,000/년
```

## PRD.md 권장 구조

```
1. 제품 정의 (한 문장)
2. 절대 제약 (SaaS X, 로컬 전용, 안전 우선)
3. 타겟 사용자 (Edge Case 페르소나 3개 + 구체적 시나리오)
4. Green Zone 기능 (5-7개, 4개 관점 모두 동의)
5. Yellow Zone 기능 (포함 조건 명시)
6. Red Zone 기능 (재검토 시점 명시)
7. 안전 원칙 (코드 레벨 경계, 승인 워크플로우)
8. UX 원칙 (3단계 자율성, diff 미리보기, 롤백)
9. 기술 스택 & 아키텍처 (v1.0 범위)
10. 수익화 전략
11. 성공 지표 (Edge Case 100명, 월 40시간 절감)
12. 명시적 제외 항목 (왜 뺐는지 이유 포함)
```
