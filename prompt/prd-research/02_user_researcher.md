# User Researcher 분석
> AI Agentic Workflow Automation System (로컬 실행)
> 조사일: 2026-04-07

---

## Executive Summary

두 가지 완전히 다른 사용자 세그먼트가 존재. **Edge Case User**는 이 제품이 없으면 업무 수행이 어려운 극단적 니치 사용자, **Mainstream User**는 대안이 있는 평균 개발자. 두 집단의 요구사항은 근본적으로 상충하므로 순차적 시장 진입 전략 필수.

---

## BRANCH A: Edge Case User (극단적 니치 사용자)

### 페르소나 1: 김재훈 (44세, 데이터 파이프라인 아키텍트)
- 일일 200개 이상 자동화 스크립트 관리
- 업무의 60%가 수동 ETL, 파일 이동, API 통합
- 현재 workaround: Python + Airflow + cron (매주 6시간 수동 작업)
- **원하는 것**: "어제 들어온 데이터, 지난달과 같은 방식으로 정제해줘" → 5분 완료

### 페르소나 2: Alex (32세, 침투 테스트 전문가)
- 거짓양성 처리에 프로젝트 시간의 80% 소요
- 고객마다 다른 보고서 형식 수동 작성
- 현재 workaround: Burp Suite + 수작업 Python 스크립트
- **원하는 것**: 스캔 결과 → 자동 필터링 → Jira 티켓 → 고객 보고서

### 페르소나 3: Dr. Park (38세, 학술 연구자)
- 체계적 문헌 고찰 500-1000개 논문 분석
- 논문 검색~데이터 추출까지 6개월 이상, 70%가 반복 작업
- 현재 workaround: Python (Scrapy) + 엑셀 수작업
- **원하는 것**: "AI 의료진단 논문 3년치 수집해서 스프레드시트로" → 3일 완료

### 필수 기능 Top 3 (없으면 절대 안 씀)

1. **로컬 파일 시스템 완전 자동 수정/생성** — SaaS 자동화 도구의 한계 극복
2. **컨텍스트 유지 + 반복 패턴 인식** — 과거 작업 이력 학습, 변형 자동 적용
3. **다단계 승인 워크플로우 + 완전 감시** — 신뢰 기반 자동화

### 성공 지표 (100명 만족 기준)
- 월 40시간 이상 절감
- 작업 자동화율 60%+
- 재작업률 5% 이하
- NPS 70 이상
- 6개월 체류율 90%

## BRANCH B: Mainstream User (평균 대중 사용자)

### 페르소나 1: 이준호 (28세, 스타트업 풀스택 개발자)
- 환경 설정, 보일러플레이트, 테스트 커버리지 반복 작업
- 현재 솔루션: GitHub Copilot으로 만족도 70%
- **원하는 것**: 프로젝트 초기 설정 자동화

### 페르소나 2: 박미선 (35세, SaaS CSM)
- Salesforce + Slack + Google Sheets에 흩어진 데이터 통합
- 현재 솔루션: Zapier + Google Sheets (만족도 65%)
- **원하는 것**: 월간 리포트 자동 생성

### 페르소나 3: 장준석 (31세, 게임 회사 QA)
- 회귀 테스트 설정, 버그 리포트 포맷, 결과 리포트 수동 정리
- 현재 솔루션: Selenium + Robot Framework
- **원하는 것**: 테스트 결과 자동 리포트

### 원하는 기능 Top 3 (있으면 쓸 만함)

1. **자주 하는 작업의 "템플릿화"** — 50-70% 자동화로도 충분
2. **쉬운 롤백** — "결과 마음에 안 들면 Ctrl+Z"
3. **최소 학습 곡선 + 기존 도구 통합** — 30분 이내 첫 실행

### 전환 장벽

1. **신뢰 부족** (최대 장벽): 96% 개발자가 AI 코드 완전 신뢰 안 함
2. **기존 워크플로우와의 충돌**: 이미 Copilot+Zapier 조합 완성됨
3. **가격 민감도**: Mainstream은 월 $5도 비쌈

## Branch A vs B 비교

| 요소 | Edge Case | Mainstream |
|------|-----------|-----------|
| 문제 | 없으면 일 안 됨 | 있으면 좋지만 대안 있음 |
| 시간 절감 | 월 40-60시간 (필수) | 월 5-15시간 (선택) |
| 정확도 요구 | 100% 필수 | 70-80% 괜찮음 |
| 신뢰도 요구 | 매우 높음 | 중간 |
| 가격 | $300+/월 가능 | $5/월도 저항 |

## UX 핵심 원칙 (PRD 반영 필요)

### 원칙 1: 신뢰 = 기초 (Trust as Foundation)
- Diff 기반 미리보기 의무화
- 위험 작업(삭제/수정)은 명시적 승인 의무
- 안전 작업(조회/생성)만 자동 진행 허용

### 원칙 2: 버전 관리 필수 내장
- 모든 AI 작업 → 자동 git commit
- 언제든 이전 상태 복원
- Audit trail 기본 지원

### 원칙 3: 권한 격리 (Least Privilege)
- 명시적 권한 요청 (파일 폴더, 환경변수, 네트워크)
- 샌드박싱/컨테이너 격리 권장

### 원칙 4: 3단계 자율성 레벨
```
Level 1: 모든 변경 전 승인 필수 (기본값)
Level 2: 안전한 작업만 자동, 삭제는 승인
Level 3: 완전 자동 + 감사 로그 + 롤백 보장
```

### 원칙 5: 짧은 학습 곡선
- Mainstream: 30분 이내 첫 실행
- Edge Case: 1시간 이내 첫 실행

## 타겟 순서 추천

**Edge Case 먼저 → 6개월 후 Mainstream 확장**

이유:
1. Edge Case는 수익성 높음 (월 $300+ 가능)
2. 명확한 요구사항 → 진정한 제품 개선 피드백
3. Mainstream에는 이미 Copilot, Zapier가 있어 경합 심함
4. 레퍼런스 케이스 구축에 유리
