---

## Executive Summary

1인 개발, 6개월 타임라인의 로컬 AI 자동화 시스템에서 기술 부채 전략은 **"안전 우선 실용주의"(Safety-First Pragmatism)** 가 최적이다. 보안 관련 코드는 처음부터 클린하게, 나머지는 실용적으로 접근하되 모든 부채를 명시적으로 기록·관리한다.

## 권장: "안전 우선 실용주의" 하이브리드

### 핵심 원칙

```
원칙 1: 보안 관련 코드는 항상 클린하게
원칙 2: 비보안 코드는 실용적 부채 허용
원칙 3: 모든 부채는 레지스터에 기록
원칙 4: 월별 리팩터링 스프린트 의무화
원칙 5: 안전성 테스트는 Day 1부터
```

### 즉시 클린하게 구현할 모듈 (타협 없음)

```python
CLEAN_FROM_DAY_ONE = [
    "safety_validator.py",    # 보안 핵심
    "command_executor.py",     # 셸 인젝션 방어
    "config.py",               # 설정 관리
    "yaml_schema.py",          # 입력 검증
]

PRAGMATIC_OK = [
    "workflow_runner.py",      # 추후 계층 분리
    "history_store.py",        # 추후 DB 마이그레이션
    "cli.py",                  # 추후 UX 개선
    "prompt_templates.py",     # 추후 외부화
]
```

## 결론

가장 피해야 할 실수 두 가지:
1. **"나중에 고치면 되겠지"** — 보안 코드에 부채를 쌓는 것
2. **"처음부터 완벽하게"** — YAGNI 위반으로 사용되지 않을 추상화에 시간 낭비

**TECHNICAL_DEBT_REGISTER.yaml 파일을 Day 1에 생성하고, 월별 리팩터링 스프린트를 캘린더에 미리 예약하는 것이 핵심이다.**
